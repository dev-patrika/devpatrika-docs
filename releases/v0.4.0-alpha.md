# Release: Dev Wiki & Semantic Search (v0.4.0-alpha)

This release implements a local vector database using Chroma DB, integrates Google GenAI embeddings (`gemini-embedding-2`), builds an automated Wiki curation pipeline to extract trending terms from daily news summaries, and introduces a unified parallel search API querying both SQL relational tables and Chroma semantic documents.

---

## 🛠️ Key Deliverables

### 1. Persistent Local Vector Store (Chroma DB)
* Persists vector records in a local directory `Backend/chroma_db` (safely ignored in `.gitignore`).
* Utilizes **`gemini-embedding-2`** from Google GenAI as the embedding model, leveraging your existing `GEMINI_API_KEY`.
* Includes fallback resilience.

### 2. Auto-Curation Pipeline (LLM Term Extractor)
* Adds a scheduled pipeline that triggers after crawling feed entries.
* Prompts the LLM (using the Groq/Gemini fallback provider) to extract new frameworks, concepts, and libraries from daily summaries.
* Automatically compiles definitions and indices them into Chroma DB.

### 3. Unified Cross-Search Endpoint (`GET /api/search`)
* Combines structured SQL keyword filters on news items and GitHub repositories with semantic cosine similarity searches on Wiki concepts.
* Matches semantic concepts (e.g. searching `"stateful multi-agent systems"` retrieves the entry `"LangGraph"` even if the exact keyword sequence isn't in the definition).

---

## 📂 Code Changes

* **`requirements.txt`**: Added `langchain-chroma` and `chromadb`.
* **`.gitignore`**: Configured to ignore `chroma_db/` from Git history.
* **`app/services/vectorstore/chroma_service.py`**: Local vector db helpers (indexing, batch sync, semantic similarity).
* **`app/services/wiki_curator/pipeline.py`**: Background term extraction and curation loop.
* **`app/routers/search.py`**: Fully implemented `/api/search` routing queries to SQLite and Chroma DB.
* **`app/services/ingestion/scheduler.py`**: Configured ingestion scheduler to trigger ingestion -> summarization -> curation sequence.
