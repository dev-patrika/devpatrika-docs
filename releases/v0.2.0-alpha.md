# Walkthrough: Ingestion Services & Document Loaders (v0.2.0-alpha)

This release implements feed ingestion and document parsing for Dev Patrika. It replaces backend API placeholders with fully database-backed endpoints, schedules background ingestion loops, and secures feed data from replication.

## Changes Made

### 1. External Dependencies
* Added core parser libraries to [requirements.txt](file:///d:/Dev-Patrika/Backend/requirements.txt):
  * `langchain-core==1.4.8` & `langchain-community==0.4.2` for document structures.
  * `requests==2.34.2` for remote REST & Firebase API calls.
  * `beautifulsoup4==4.15.0` & `lxml==6.1.1` for scraping/parsing.

### 2. Custom LangChain Loaders
* Created 4 custom loaders inheriting from LangChain's `BaseLoader` that lazily stream parsed documents:
  * [hn.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/hn.py): Polls top 20 story IDs from HN Firebase API.
  * [devto.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/devto.py): Pulls feed items across various tags from Dev.to API.
  * [arxiv.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/arxiv.py): Parses Atom XML feeds for cs.AI, cs.LG, and cs.SE preprints.
  * [github_trending.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/github_trending.py): Scrapes the daily trending page on GitHub using BeautifulSoup.

### 3. Ingestion Helpers & Orchestrator
* Created [helpers.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/helpers.py):
  * Formats time offsets into relative relative-freshness strings (e.g. "3 hours ago", "Yesterday").
  * Rules-based keyword mapping engine to auto-classify titles/descriptions into predefined developer categories.
  * Token-based Jaccard similarity scorer (`check_title_similarity`) to detect repeated articles.
* Created [orchestrator.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/orchestrator.py):
  * Coordinates all loaders.
  * Prevents duplicate URLs from entering the database.
  * Filters out items that share >80% title similarity with stories ingested in the last 24 hours.

### 4. Background Job Scheduling
* Created [scheduler.py](file:///d:/Dev-Patrika/Backend/app/services/ingestion/scheduler.py):
  * Runs a continuous, non-blocking `asyncio` task loop inside FastAPI.
  * Polling is scheduled to execute every 6 hours.
* Modified [main.py](file:///d:/Dev-Patrika/Backend/app/main.py):
  * Invokes the background scheduler on application boot inside the startup lifecycle event.

### 5. API Endpoints
* Modified [news.py](file:///d:/Dev-Patrika/Backend/app/routers/news.py):
  * `GET /api/news`: Retrieves news items from SQLite, sorting by newest publications and supporting `category`, search keyword `q`, and page `limit` filters.
  * `POST /api/news/ingest`: Dispatches a manual run of the crawler asynchronously using FastAPI `BackgroundTasks`.
* Modified [github.py](file:///d:/Dev-Patrika/Backend/app/routers/github.py):
  * `GET /api/github/trending`: Queries and returns stored repositories from SQLite sorted by stars.
