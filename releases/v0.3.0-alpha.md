# Walkthrough: LangChain Core Processing & Tools (v0.3.0-alpha)

This release implements LLM processing pipelines using LangChain to automatically clean, summarize, and categorize tech news articles and GitHub repositories. It also sets up custom semantic search tools and automated Wiki page compilation.

## Changes Made

### 1. External Dependencies
* Updated [requirements.txt](file:///d:/Dev-Patrika/Backend/requirements.txt):
  * Added `langchain-google-genai` and `langchain-groq` for Gemini and Groq model APIs.
  * Upgraded `pydantic-settings==2.10.1` to resolve dependency conflicts with `langchain-community`.

### 2. LLM Failover & Fallback Provider
* Created [llm.py](file:///d:/Dev-Patrika/Backend/app/services/processing/llm.py):
  * Exposes `get_llm(temperature=0.0)` returning `ChatGroq` (`openai/gpt-oss-120b`) as primary LLM.
  * Configures `ChatGoogleGenerativeAI` (`gemini-2.5-flash`) as the fallback list using LangChain's native `.with_fallbacks()` mechanism.

### 3. News & GitHub Analysis Pipelines
* Created [pipeline.py](file:///d:/Dev-Patrika/Backend/app/services/processing/pipeline.py):
  * Implements `NewsAnalysis` and `GitHubAnalysis` structured outputs using Pydantic.
  * Formats AI outputs into consistent, premium markdown blocks containing:
    * **Overview** (what it is/how it works)
    * **Key Details** (structured bullet points)
    * **Community & Traction** (popularity/developer adoption)
  * Implements `process_pending_items(session)` which queries SQLite for pending news items or GitHub repositories without summaries, runs the LLM analysis, updates the columns, and commits changes in batches.

### 4. Dev Wiki Generator Agent
* Created [wiki_generator.py](file:///d:/Dev-Patrika/Backend/app/services/processing/wiki_generator.py):
  * Exposes `generate_wiki_definition(term, session)`:
    * Asks the LLM to write a markdown definition, trending details, and official documentation reference links for any term.
    * Stores or updates the wiki details inside the `wiki_entries` table.

### 5. Custom LangChain Search & Utility Tools
* Created [search_tools.py](file:///d:/Dev-Patrika/Backend/app/tools/search_tools.py):
  * `news_search_tool`: Queries news items from the local SQLite database.
  * `github_search_tool`: Uses the public GitHub search API to find repos or issues.
  * `url_summarizer_tool`: Scrapes web content using BeautifulSoup, cleans tags, splits text, and yields a summary of the external URL on-the-fly.
* Registered search tools in [__init__.py](file:///d:/Dev-Patrika/Backend/app/tools/__init__.py).

### 6. Router & Endpoint Updates
* Modified [news.py](file:///d:/Dev-Patrika/Backend/app/routers/news.py):
  * Updated `POST /api/news/ingest` to execute feed ingestion followed by background LLM summarization/classification.
  * Added `POST /api/news/process` to trigger the AI processing task manually.
* Modified [wiki.py](file:///d:/Dev-Patrika/Backend/app/routers/wiki.py):
  * Exposed `GET /api/wiki` for listing/searching stored wiki terms.
  * Exposed `GET /api/wiki/{term}` supporting case-insensitive DB term lookups.
  * Added `POST /api/wiki/generate` to manually request LangChain to generate a wiki page entry in the background.
* Modified [wiki_schema.py](file:///d:/Dev-Patrika/Backend/app/schemas/wiki_schema.py):
  * Added Pydantic `@field_validator` to automatically deserialize the JSON-serialized string of links stored in SQLite into a list format when querying.
