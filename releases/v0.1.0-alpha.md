# Release Notes: Dev Patrika Backend `v0.1.0-alpha`

This release establishes the baseline framework, modular file architecture, database configuration, tables, and schemas for the **Dev Patrika** Developer Intelligence Platform backend.

## 🚀 Key Deliverables

### 📂 1. Modular Directory Structure
Initialized modular packaging layout to support incremental expansion through Phase 1:
- `app/core/`: Constants (Categories, News sources), uniform logger configuration, and central exception handler.
- `app/models/`: Declarative SQLModel tables (`news_items`, `wiki_entries`, `github_radar`, `personal_notes`).
- `app/schemas/`: Pydantic validation schemas for news, wiki entries, GitHub trending repositories, and chat responses.
- `app/routers/`: Configured API endpoints including a liveness `/api/health` router and route placeholders for other spec endpoints.
- `app/services/` & `app/tools/` & `app/agents/`: Empty placeholder directories to separate business ingestion logic, custom tools, and agent workflows.

### ⚙️ 2. Configuration & Utilities
- Configured environment variable loading using Pydantic Settings from a local `.env` configuration file.
- Suppressed protected namespace warning triggers for Pydantic V2 on the `/api/ai/chat` schemas.

### 🛢️ 3. Relational Storage
- Initialized SQLite database connection engine in `app/database.py` with multi-thread session management.
- Configured lifespan startup event to automatically create structured tables (`NewsItem`, `WikiEntry`, `GitHubRadar`, `PersonalNote`) inside `dev_patrika.db`.
